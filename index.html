StreamCloud

Plataforma de streaming distribuida inspirada en Netflix, diseñada para ofrecer contenido multimedia de forma rápida, escalable, disponible y tolerante a fallos.

Ver prototipo

Introducción

StreamCloud es una propuesta de plataforma de streaming distribuida que permite a los usuarios acceder a películas, series y contenido multimedia desde diferentes dispositivos conectados a internet.

A diferencia de una plataforma centralizada, StreamCloud divide sus funciones en varios servicios conectados entre sí, como usuarios, catálogo, reproducción, recomendaciones, almacenamiento y notificaciones.

Objetivos del proyecto

Objetivo general

Diseñar una plataforma de streaming distribuida inspirada en Netflix, que permita ofrecer contenido multimedia de manera eficiente, escalable, disponible y tolerante a fallos.

Objetivos específicos

  • Describir el funcionamiento de una plataforma de streaming distribuida.
  • Identificar problemas de un sistema centralizado.
  • Proponer una arquitectura basada en microservicios.
  • Aplicar escalabilidad, replicación y disponibilidad.
  • Presentar una interfaz visual del sistema.

Descripción del sistema

El sistema permite que los usuarios puedan crear una cuenta, iniciar sesión, seleccionar un perfil, revisar un catálogo de contenido, buscar películas o series, recibir recomendaciones y simular la reproducción de videos.

Problemática actual

En una plataforma centralizada, gran parte del funcionamiento depende de un solo servidor. Esto puede causar lentitud, errores o caídas cuando muchos usuarios intentan acceder al sistema al mismo tiempo.

Servidor saturado

Un solo servidor recibe demasiadas solicitudes.

Punto único de falla

Si falla el servidor principal, toda la plataforma puede dejar de funcionar.

Difícil escalabilidad

Agregar recursos puede ser complicado y costoso.

Arquitectura del sistema

StreamCloud utiliza una arquitectura distribuida basada en clientes, balanceador de carga, API Gateway, microservicios, bases de datos replicadas, almacenamiento en la nube y CDN.

Características distribuidas implementadas

Escalabilidad

Permite agregar más servidores cuando aumenta la cantidad de usuarios.

Tolerancia a fallos

El sistema puede seguir funcionando aunque un servicio presente errores.

Replicación

La información importante se mantiene copiada en varias bases de datos.

Comunicación entre nodos

Los microservicios se comunican mediante API Gateway y conexiones internas.

Disponibilidad

La plataforma busca estar accesible la mayor parte del tiempo.

Cache distribuido

Guarda información frecuente para responder más rápido.

Conclusión

StreamCloud demuestra cómo una plataforma de streaming puede beneficiarse de los sistemas distribuidos. Gracias a su arquitectura basada en microservicios, replicación, balanceo de carga y disponibilidad, el sistema puede ofrecer una experiencia más estable, rápida y preparada para crecer.
